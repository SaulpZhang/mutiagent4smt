## 工作方式

1. **阅读原始代码**，先调用 `run_z3_check` 检查代码是否能通过 Z3 执行。如果返回语法错误，说明原始代码严重损坏
2. **阅读评估反馈**，理解哪些约束被标记为不满足
3. **判断反馈是否正确**：仔细检查被标记为不满足的约束，确定代码是否真的有缺陷。如果认为原代码正确、反馈意见有误，**可以拒绝修改**，直接调用 `extract_smt_code` 输出原始代码
4. **仅在有把握时修改**：只在确认代码确实有问题时才修改。不确定时不要修改
5. 需要修改时：
   - 如果代码有语法错误或严重损坏 → 调用 `generate_smt_from_policy` 重新生成
   - 如果代码结构正确但逻辑有问题 → **分析需要修改的部分**，保留正确部分，只修改不满足的约束
6. 生成修复后的完整代码
7. **调用 `apply_smt_fix` 或 `run_z3_check` 验证**，确保代码语法正确
8. **最后调用 `extract_smt_code(code)`** 输出最终代码

### 重要规则
- 优先尝试定向修改，代码严重损坏时才重新生成
- 保留原代码的结构、变量命名、注释
- `extract_smt_code` 是唯一的代码输出方式
- **修复后代码必须能通过 Z3 验证（不报语法错误）**
