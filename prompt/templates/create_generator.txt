你是一个SMT-LIB V2代码生成器（Generator）的创建专家。你的任务是创建一个Python函数，该函数能根据IAM配置和约束列表自动生成可执行的SMT-LIB V2代码。

## Generator 规范

### 函数签名

```python
    Returns:
        可执行的SMT-LIB V2代码字符串
    """
    # 你的实现
```

### 输出格式要求

你必须输出严格的JSON格式，包含以下三个字段：

1. `generator_name`: 生成器名称（英文小写+下划线，如 `bucket_operator_condition`）
2. `spec`: 生成器规格说明（字典格式）：
   - `name`: 同上
   - `version`: "1.0.0"
   - `description`: 简要说明适用场景
   - `input`: 输入描述
   - `output`: 输出描述
   - `examples`: 匹配的验证指令示例（数组）
3. `generator_code`: Python代码字符串（def generate 函数的完整实现）

### 背景知识

IAM配置采用**华为云IAM格式**（Version 5.0），涉及OBS存储桶策略和委托信任策略。
- Principal类型为 `"IAM"`（如 `"Principal": {"IAM": ["account1"]}`）
- Action使用华为云命名（如 `sts:agencies:assume`、`obs:object:listBucket`）
- Condition操作符包括 `stringequals`、`stringmatch`、`forallvalues:stringequalsnot`、`foranyvalue:stringmatch`、`numericequals`、`null`、`dateequals`、`ipaddress` 等
- Condition键使用 `g:` 前缀（如 `g:PrincipalType`, `g:SourceIp`）
- Effect值必须是 `Allow` 或 `Deny`

### SMT-LIB V2 语法规则

1. 变量先声明后使用：`(declare-const name Bool)` / `(declare-const name String)`
2. 注释用 `;`，逻辑运算用 `and`/`or`/`not`/`=>`，小写 true/false
3. 必须含 `(check-sat)` 和 `(exit)`，括号严格匹配
4. **字符串禁止使用 `\"`**（SMT-LIB V2不支持反斜杠转义），直接写纯文本如 `"Allow"`
5. **禁止将嵌套JSON编码为含引号的字符串** → 拆分为独立变量

### JSON分解规则示例

```
IAM配置                    →  SMT变量
Principal: {"IAM": ["a"]}  →  principal_type="IAM", principal_value="a"
Principal: {"ID": ["*"]}   →  principal_type="ID", principal_value="*"
Condition: {"stringequals": {"g:PrincipalType": ["User"]}}
                           →  condition_operator="stringequals", condition_key="g:PrincipalType", condition_value="User"
Action: ["listBucket"]     →  action_value="listBucket"
Condition: {}（空对象）     →  has_condition=false
```

### Generator 代码规范

1. 必须导入所需模块（json 等标准库）
2. `generate()` 函数必须是纯Python，不含外部依赖
3. 从 `config` 字典中提取IAM策略字段（可能含 `bucket_policy` 或 `trust_policy` JSON字符串，需要用 `json.loads()` 解析）
4. 根据 `constraints` 生成对应的SMT断言
5. 生成的SMT代码需同时处理 Allow 和 Deny 的Effect
6. 返回的字符串必须是合法SMT-LIB V2代码
7. 代码中不要用 f-string 拼接SMT代码块（使用字符串变量拼接或列表+join）

### generator_code 示例结构

```python
def generate(config: dict, constraints: list) -> str:
    # 1. 解析配置
    bucket_policy_str = config.get("bucket_policy", "{}")
    if isinstance(bucket_policy_str, str):
        import json
        policy = json.loads(bucket_policy_str)

    # 2. 提取Statement
    statements = policy.get("Statement", [])

    # 3. 生成SMT变量声明
    lines = []
    for i, stmt in enumerate(statements):
        effect = stmt.get("Effect", "")
        lines.append(f"(declare-const effect_{i} String)")
        lines.append(f"(assert (= effect_{i} \"{effect}\"))")
        # ... 更多变量

    # 4. 根据约束生成断言
    for c in constraints:
        # ... 映射约束到SMT断言

    # 5. 返回完整代码
    lines.append("(check-sat)")
    lines.append("(exit)")
    return "\\n".join(lines)
```

### config 常见结构

存储桶策略（bucket_policy）：
```json
{
  "bucket_policy": "{\"Version\": \"5.0\", \"Statement\": [{\"Effect\": \"Allow\", \"Principal\": {\"IAM\": [\"account1\"]}, \"Action\": [\"obs:object:listBucket\"], \"Condition\": {...}}]}"
}
```

委托信任策略（trust_policy）：
```json
{
  "trust_policy": "{\"Version\": \"5.0\", \"Statement\": [{\"Effect\": \"Allow\", \"Principal\": {\"IAM\": [\"account1\"]}, \"Action\": [\"sts:agencies:assume\"], \"Condition\": {...}}]}"
}
```

注意：bucket_policy 和 trust_policy 是 JSON 字符串，需要用 `json.loads()` 解析后再使用。
