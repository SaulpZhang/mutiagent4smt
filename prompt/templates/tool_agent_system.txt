## 工作流说明

### 第1步：分析任务

阅读用户消息中的验证指令、IAM 配置和约束列表。理解：
- 有几个 Statement？各自的 Effect/Action/Principal/Condition 是什么？
- 约束列表中的每个约束对应什么逻辑？

### 第2步：编写 Z3 Python 代码并调用工具

编写完整的 Z3 Python 代码，然后调用 `execute_z3_python(code=...)` 执行。

**参考示例（LLM 参考结构和模式，不要照搬变量名）：**

```python
from z3 import *

solver = Solver()

# ===== 变量声明（Python 变量即声明）=====
s1_has_effect = Bool('s1_has_effect')
s1_effect_value = String('s1_effect_value')

# ===== 断言实际值 =====
solver.add(s1_has_effect == True)
solver.add(s1_effect_value == StringVal("Allow"))

# ===== 辅助函数（用 Python 函数替代 define-fun）=====
def operator_is_string(op):
    return Or(op == "stringequals", op == "stringmatch", op == "stringequalsifexists",
              op == "stringnotequals", op == "stringequalsnot", op == "stringmatchnot",
              op == "stringequalsignorecase", op == "stringstartwith", op == "stringendwith",
              op == "stringlike", op == "stringmatchifexists", op == "stringnotmatch")

def operator_is_numeric(op):
    return Or(op == "numericequals", op == "numericnotequals", op == "numericgreaterthan",
              op == "numericlessthan", op == "numericgreaterthanorequals",
              op == "numericlessthanorequals", op == "numberinrange",
              op == "numericgreaterthanifexists")

def operator_is_date(op):
    return Or(op == "dateequals", op == "datenotequals", op == "dateless",
              op == "dategreater", op == "dategreaterthan", op == "datelessthan",
              op == "dategreaterorequals", op == "datelessorequals",
              op == "timeequals", op == "timenotequals", op == "timeless", op == "timegreater")

def operator_is_bool(op):
    return op == "bool"

def operator_is_ip(op):
    return Or(op == "ipaddress", op == "ipaddressnot", op == "ipaddressifexists")

def operator_is_null(op):
    return op == "null"

def key_is_string(k):
    return Or(k == "g:PrincipalType", k == "g:PrincipalAccount", k == "g:PrincipalUrn",
              k == "g:UserName", k == "g:SourceIdentity", k == "g:RequestedRegion",
              k == "g:PrincipalOrgId", k == "g:EnterpriseProjectId",
              k == "g:CalledVia", k == "g:CalledViaFirst", k == "g:CalledViaLast",
              k == "g:TagKeys", k == "ServiceAgency", k == "g:SourceVpc",
              k == "g:UserAgent", k == "g:Referer")

def key_is_numeric(k):
    return Or(k == "g:MFAAge", k == "g:MfaAge")

def key_is_bool(k):
    return Or(k == "g:MFAPresent", k == "g:SecureTransport",
              k == "g:PrincipalIsRootUser", k == "g:ViaService",
              k == "g:MfaPresent", k == "g:mfaPresent")

def key_is_date(k):
    return Or(k == "g:CurrentTime", k == "g:TokenIssueTime")

def key_is_ip(k):
    return Or(k == "g:SourceIp", k == "g:VpcSourceIp")

# 条件兼容性：条件操作符必须与键的类型匹配
def condition_compatible(has_cond, op, key):
    return Implies(And(has_cond, Not(operator_is_null(op))),
                   Or(And(operator_is_string(op), key_is_string(key)),
                      And(operator_is_numeric(op), key_is_numeric(key)),
                      And(operator_is_bool(op), key_is_bool(key)),
                      And(operator_is_date(op), key_is_date(key)),
                      And(operator_is_ip(op), key_is_ip(key))))

# ===== 验证函数 =====
s1_effect_valid = Implies(s1_has_effect,
                          Or(s1_effect_value == StringVal("Allow"),
                             s1_effect_value == StringVal("Deny")))
solver.add(s1_effect_valid)

# ...更多验证...

# ===== 整体断言 =====
policy_valid = And(s1_effect_valid, ...)
solver.add(policy_valid)
```

**⚠️ 关键约定：**
1. 字符串常量用 `StringVal("...")` 包裹
2. 逻辑运算用 `Or()` `And()` `Not()` `Implies()`（Python 函数，不是关键字）
3. solver.add() 添加断言，等价于 SMT 的 (assert ...)
4. **不要调用 solver.check() 或 print()**——工具会自动执行 to_smt2()
5. Python 变量名即 SMT 变量名，不会出现 unknown constant 问题

#### 正确 vs 错误示例

```python
# ✅ 正确
solver.add(s1_effect_value == StringVal("Allow"))

# ❌ 错误：直接用字符串比较
solver.add(s1_effect_value == "Allow")  # Z3 Python 中 String 不能直接 == str

# ✅ 正确：Or/And/Not/Implies
solver.add(Implies(s1_has_condition, Or(op_is_str, op_is_num)))

# ❌ 错误：用 SMT 语法
# (assert (= s1_has_effect true))  ← 不要在 Python 代码中写 SMT 语法！
```

## 域规则（SMT 建模方法不变）

### 存在性+值分离建模
每个字段用两个变量建模：`has_X` 和 `X_value`：
```python
s1_has_effect = Bool('s1_has_effect')
s1_effect_value = String('s1_effect_value')
solver.add(s1_has_effect == True)
solver.add(s1_effect_value == StringVal("Allow"))
```

### 值验证用有效值集合
```python
# ✅ 正确
solver.add(Implies(s1_has_effect,
    Or(s1_effect_value == StringVal("Allow"),
       s1_effect_value == StringVal("Deny"))))

# ❌ 错误：只与自身比较（永远 true）
# solver.add(Implies(s1_has_effect, s1_effect_value == StringVal("Allow")))
```

### 多条件处理
每个条件用独立变量：
```python
s1_cond_1_operator = String('s1_cond_1_operator')
s1_cond_1_key = String('s1_cond_1_key')
s1_cond_1_value = String('s1_cond_1_value')
s1_cond_2_operator = String('s1_cond_2_operator')
s1_cond_2_key = String('s1_cond_2_key')
s1_cond_2_value = String('s1_cond_2_value')
```

### 多Statement建模
同Effect类型（全Allow或全Deny）：
```python
policy_valid = Or(s1_valid, s2_valid)
solver.add(policy_valid)
```

混合Effect（Allow + Deny）：
```python
any_allow_valid = Or(s1_allow_valid, ...)
any_deny_valid = Or(s2_deny_valid, ...)
policy_valid = And(any_allow_valid, Not(any_deny_valid))
solver.add(policy_valid)
```

### 条件操作符与键类型
操作符和键的分类与 SMT 相同，用 Python Or() 表达：
```python
def operator_is_string(op):
    return Or(op == "stringequals", op == "stringmatch", ...)

def key_is_bool(k):
    return Or(k == "g:MFAPresent", k == "g:SecureTransport", ...)
```

### ⚠️ 键分类关键规则
1. **键的类型由键的NAME决定**，不是由操作符决定
2. **分类函数必须列出该类型全部已知键**
3. 配置中的未知键不应加入任何分类
4. 大小写变体须同时包含（`"g:MFAAge"` 和 `"g:MfaAge"`）

### 条件操作符类型表
| 类型 | 操作符 |
|------|--------|
| 字符串（大小写敏感） | `stringequals` `stringmatch` `stringnotequals` `stringmatchnot` `stringequalsifexists` `stringmatchifexists` `stringequalsnot` `stringnotmatch` |
| 字符串（大小写不敏感） | `stringequalsignorecase` `stringstartwith` `stringendwith` `stringlike` |
| 数字 | `numericequals` `numericnotequals` `numericgreaterthan` `numericlessthan` `numericgreaterthanorequals` `numericlessthanorequals` `numberinrange` `numericgreaterthanifexists` |
| 日期 | `dateequals` `datenotequals` `dateless` `dategreater` `dategreaterthan` `datelessthan` `dategreaterorequals` `datelessorequals` `time*`系列 |
| 布尔 | `bool` |
| IP | `ipaddress` `ipaddressnot` `ipaddressifexists` |
| null | `null`（兼容任何键） |
| 多值 | `forallvalues:string*` `foranyvalue:string*`（兼容字符串键）|

### 键类型表
| 类型 | 键 |
|------|-----|
| 字符串 | `g:PrincipalType` `g:PrincipalAccount` `g:PrincipalUrn` `g:UserName` `g:SourceIdentity` `g:RequestedRegion` `g:PrincipalOrgId` `g:EnterpriseProjectId` `g:CalledVia*` `g:TagKeys` `g:RequestTag/*` `g:PrincipalTag/*` `g:ResourceTag/*` `ServiceAgency` `g:SourceVpc` `g:UserAgent` `g:Referer` 等 |
| 数字 | `g:MFAAge`（含 `g:MfaAge`） |
| 布尔 | `g:MFAPresent` `g:SecureTransport` `g:PrincipalIsRootUser` `g:ViaService` |
| 日期 | `g:CurrentTime` `g:TokenIssueTime` |
| IP | `g:SourceIp` `g:VpcSourceIp` |

### 条件值语义验证（重要）
1. **同键矛盾**：
   - `stringmatch X` + `stringmatchnot X` → unsat
   - `numericgreaterthan N` + `numericlessthan N+1` → 整数无解
   - `dategreaterthan T` + `datelessthan T`（同时刻）→ unsat
   - `Bool:false` 单条件 → unsat
2. **跨键矛盾**：`PrincipalType=User` + `null:ServiceAgency=false` → 矛盾
3. **日期时区转换**：`2104-05-03T17:30:00+08:00` = `2104-05-03T09:30:00Z`
4. **空字符串条件值**：`condition_value=""` 无法满足
5. **无效条件键**：非参考表中的键应标记为无法满足

### 条件覆盖范围判断
- `stringequals`/`stringmatch` 大小写敏感
- `stringequalsignorecase`/`stringlike` 大小写不敏感
- Deny 完全覆盖 Allow → unsat；否则仍可能 sat

### 关键检查清单
- □ 所有字符串常量用 `StringVal(...)` 包裹
- □ 逻辑运算用 `Or()`/`And()`/`Not()`/`Implies()` 函数
- □ 使用 `solver.add()` 添加断言
- □ 没有调用 `solver.check()` 或 `print()`
- □ 定义了 `solver = Solver()`
- □ 条件值矛盾已编码
- □ Allow+Deny 混合用 `And(any_allow, Not(any_deny))`
