## 工作流说明

### 第1步：调用 smt_declare_and_assign

调用后你会收到一段 SMT 代码，包含所有变量的 `declare-const` 声明和 `assert` 赋值。例如：

```
(declare-const s1_has_effect Bool)
(declare-const s1_effect_value String)
(assert (= s1_has_effect true))
(assert (= s1_effect_value "Allow"))
```

**仔细阅读这段输出**——后面的代码必须基于这里声明的变量名来编写，变量名必须完全匹配。

### 第2步：根据验证指令 + 约束列表 + 变量声明 编写剩余代码

`smt_declare_and_assign` 只完成了"变量声明和赋值"。你需要结合以下三方面信息，**直接在回答中输出**完整的 SMT-LIB V2 代码：

1. **原始验证指令**（任务描述）—— 告诉你验证目标
2. **约束列表**（JSON）—— 告诉你需要逐项验证什么
3. **第1步的变量声明**—— 告诉你可以用哪些变量

你需要定义：
- `define-fun` — 定义验证函数（引用第1步声明的变量）
- `assert` — 断言验证条件
- `check-sat` — 求解
- `exit` — 退出

### 第3步：调用 smt_verify

传入完整的 SMT 代码（第1步的声明 + 第2步的验证逻辑），检查语法正确性。如果检查不通过，修正后重试。

## SMT-LIB V2 语法规则

1. 变量先声明后使用：(declare-const name Bool) / (declare-const name String)
2. 注释用 ;，逻辑运算用 and/or/not/=>，小写 true/false
3. 必须含 (check-sat) 和 (exit)，括号严格匹配
4. 字符串禁止使用 \"，直接写纯文本如 "Allow"
5. 禁止将嵌套 JSON 编码为含引号的字符串 → 拆分为独立变量
