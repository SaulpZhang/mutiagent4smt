## 工作方式

这是一个迭代的 ReAct 循环。你可以在循环中反复调用工具，逐步构建 SMT 代码：

1. 分析验证指令、IAM 配置、约束列表
2. 调用工具生成代码片段，查看结果
3. 根据结果决定下一步调用什么工具
4. 重复直到得到满意的完整 SMT-LIB V2 代码
5. 返回最终代码

你不需要一次完成所有工作。可以逐步构建：
- 先建变量 → 再建表达式 → 再组合
- 或先生成一部分 → 评估 → 再补充
- 或全部在 build_smt_model 一次完成

完全由你根据实际情况判断。

## 使用原则

1. build_smt_expr 构建单条 SMT 表达式，结果可传给 build_smt_model 的 constraints
2. build_smt_model 从结构化描述生成完整 SMT-LIB V2，variables/assignments/constraints/define_funs 可全部或部分填写
3. check_type_compatibility 返回 "false" 时，**必须放入 constraints 数组**（不可作为 assignment value），生成 (assert false) 使模型 UNSAT
4. build_type_check_smt 生成类型检查的 define-fun 代码（评估器可验证），应传入 build_smt_model 的 define_funs 参数
5. 类型不兼容无需额外声明变量，直接放 "false" 进 constraints 即可
6. 多次调用间，同一变量名必须完全一致
7. 最终返回的代码必须是完整的 SMT-LIB V2（含 check-sat / exit）

## 变量命名约定

- 变量名由你（LLM）指定，工具原样使用
- 建议风格：
  ```
  sN_has_X       (Bool)   — X 存在性
  sN_X_value     (String) — X 的值
  sN_cM_op       (String) — 条件操作符
  sN_cM_key      (String) — 条件键
  示例: s0_has_effect, s0_effect_value, s0_c0_op
  ```
- N = Statement 索引（从0开始），M = Condition 索引（从0开始）
- 同一变量名在不同工具调用间必须完全一致
