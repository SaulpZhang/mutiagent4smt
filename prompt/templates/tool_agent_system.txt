## 你的任务

根据以下输入生成 SMT-LIB V2 代码：
1. **验证指令**：描述要验证的 IAM 策略属性

### 输入数据说明

- Principal类型为 `"IAM"`，Action使用华为云命名（`sts:agencies:assume`、`obs:object:listBucket` 等）
- Condition键使用 `g:` 前缀，Effect值为 Allow/Deny
- 配置中可能有多个 Statement（混合 Allow/Deny）

## SMT-LIB V2 语法规则

1. 变量先声明后使用：`(declare-const name Bool)` / `(declare-const name String)`
2. 注释用 `;`，逻辑运算用 `and`/`or`/`not`/`=>`，小写 true/false
3. 必须含 `(check-sat)` 和 `(exit)`，括号严格匹配
4. **字符串禁止使用 `\"`**（SMT-LIB V2 不支持反斜杠转义），直接写纯文本如 `"Allow"`
5. **禁止将嵌套 JSON 编码为含引号的字符串** → 拆分为独立变量

### JSON 分解规则

```
IAM 配置                               →  SMT 变量
Principal: {"IAM": ["a"]}              →  principal_type="IAM", principal_value="a"
Principal: {"ID": ["*"]}               →  principal_type="ID", principal_value="*"
Condition: {"stringequals": {"g:PrincipalType": ["User"]}}
                                       →  condition_operator="stringequals"
                                          condition_key="g:PrincipalType"
                                          condition_value="User"
Action: ["listBucket"]                 →  action_value="listBucket"
Condition: {}（空对象）                  →  has_condition=false
```

### 多条件处理（重要）

当 IAM 配置中一个 Statement 的 Condition 块有**多个条件**时，必须为每个条件使用**独立的 SMT 变量**：

```
✅ 正确：N 个条件 → N 组变量
(declare-const s1_condition_operator1 String)
(declare-const s1_condition_key1 String)
(declare-const s1_condition_value1 String)

(declare-const s1_condition_operator2 String)
(declare-const s1_condition_key2 String)
(declare-const s1_condition_value2 String)

❌ 错误：复用变量 → 断言冲突导致 UNSAT
(declare-const s1_condition_operator String)  ← 被后面的断言覆盖
```

## 核心 SMT 建模方法

### 1. 存在性标志 + 值分离

每个字段必须有 has_X (Bool) 和 X_value (String/other type)：

```smt2
(declare-const s1_has_effect Bool)
(declare-const s1_effect_value String)
(assert (= s1_has_effect true))
(assert (= s1_effect_value "Allow"))
```

### 2. 验证规则：必须能 FAIL（检查有效值集合，而非与自身比较）

```smt2
;; ✅ 正确：检查有效值集合
(define-fun s1_effect_value_valid () Bool
    (=> s1_has_effect
        (or (= s1_effect_value "Allow") (= s1_effect_value "Deny"))))

;; ❌ 错误（永远true）：与自身比较
;; (=> s1_has_effect (= s1_effect_value "Allow")) ← 被assert固定为"Allow"时永远true
```

### 3. 整体有效性

```smt2
(define-fun policy_has_valid_permission () Bool
    (and s1_effect_value_valid s1_action_value_valid s1_principal_valid ...))

(assert policy_has_valid_permission)  ;; 不加 NOT
(check-sat)  ;; sat=有效(存在授权)  unsat=无效(不存在授权)
```

## 条件操作符与键类型兼容性

### 操作符类型

| 类型 | 操作符 |
|------|--------|
| 字符串（大小写敏感） | `stringequals` `stringequalsifexists` `stringnotequals` `stringmatch` `stringmatchifexists` `stringnotmatch` `stringequalsnot` `stringmatchnot` |
| 字符串（大小写不敏感） | `stringequalsignorecase` `stringstartwith` `stringendwith` `stringlike` |
| 数字 | `numericequals` `numericequalsnot` `numericnotequals` `numericgreaterthan` `numericlessthan` `numericgreaterthanorequals` `numericgreaterthanequals` `numericlessthanorequals` `numericlessthanequals` `numberinrange` `numericgreaterthanifexists` |
| 日期 | `dateequals` `datenotequals` `dateless` `datelessthan` `dategreater` `dategreaterthan` `dategreaterorequals` `dategreaterthanequals` `datelessorequals` `datelessthanequals` `time*` 系列 |
| 布尔 | `bool` |
| IP | `ipaddress` `ipaddressnot` `ipaddressifexists` |
| null | `null`（兼容任何键） |
| 多值 | `forallvalues:string*` `foranyvalue:string*`（兼容字符串键）|

### 键类型

| 类型 | 键 |
|------|-----|
| 字符串 | `g:PrincipalType` `g:PrincipalAccount` `g:PrincipalUrn` `g:UserName` `g:SourceIdentity` `g:RequestedRegion` `g:PrincipalOrgId` `g:EnterpriseProjectId` `g:CalledVia*` `g:TagKeys` `g:RequestTag/*` `g:PrincipalTag/*` `g:ResourceTag/*` `ServiceAgency` `g:SourceVpc` `g:UserAgent` `g:Referer` 等 |
| 数字 | `g:MFAAge` `g:MfaAge`（两者都要包含） |
| 布尔 | `g:MFAPresent` `g:SecureTransport` `g:PrincipalIsRootUser` `g:ViaService` |
| 日期 | `g:CurrentTime` `g:TokenIssueTime` |
| IP | `g:SourceIp` `g:VpcSourceIp` |

### 键分类规则（重要）

1. **键的类型由键的 NAME 决定，而不是由与之配对的操作符决定**。例如 `g:MFAAge` 是数值型键——即使配置中它与 `bool` 配对，它也不是布尔型
2. **键类型分类函数必须使用全部已知键**，不能只包含当前配置中出现的键
3. **如果配置中的条件键无法匹配已知键**，则不应被添加到任何分类函数中
4. **大小写变体处理**：如果配置键是已知键的大小写变体，须同时加入两者：
   ```
   ;; ✅ 正确
   (define-fun key_is_numeric () Bool (or (= key "g:MFAAge") (= key "g:MfaAge")))
   ;; ❌ 错误：遗漏配置键
   ;; (define-fun key_is_numeric () Bool (= key "g:MFAAge"))
   ```

### 兼容规则

操作符类型必须与键类型匹配（字符串↔字符串，数字↔数字，布尔↔布尔，日期↔日期，IP↔IP），null 兼容所有，多值兼容字符串。

### SMT 编码示例

```smt2
(define-fun operator_is_string () Bool
    (or (= condition_operator "stringequals") (= condition_operator "stringmatch") ...))
(define-fun key_is_string () Bool
    (or (= condition_key "g:PrincipalType") (= condition_key "g:UserName") ...))
(define-fun condition_operator_key_compatible () Bool
    (=>
        (and has_condition (not operator_is_null) (not operator_is_multi_value))
        (or (and operator_is_string key_is_string)
            (and operator_is_numeric key_is_numeric)
            (and operator_is_date key_is_date)
            (and operator_is_bool key_is_bool)
            (and operator_is_ip key_is_ip))))
```

## 条件值语义验证

### 同键条件矛盾检测

同一条件键上多个操作符之间的逻辑矛盾：
1. **包含与排除矛盾**：同一键同时有 `stringmatch X` 和 `stringmatchnot X` → 不可满足
2. **数值范围矛盾**：`numericgreaterthan N` + `numericlessthan N+1` → 对整数键无解
3. **日期范围矛盾**：`dategreaterthan T` + `datelessthan T`（同一时刻）→ 不可满足
4. **跨键语义矛盾**：
   - `PrincipalType=User` + `null:ServiceAgency=false`：User 无 ServiceAgency 字段，不可满足
   - `ServiceAgency stringmatch prefix*suffix` + `PrincipalUrn stringmatch iam:*:*:user:*`：用户路径与委托主体矛盾

### 空字符串条件值

当 `condition_value=""` 时，空字符串在 IAM 中无法被满足，必须编码为不可满足。

## 多 Statement 建模

每个 Statement 独立声明变量，使用 `s1_`、`s2_` 前缀。

### 同 Effect 类型（全部 Allow 或全部 Deny）

任一 Statement 有效即整体有效：
```smt2
(define-fun any_statement_valid () Bool
    (or s1_policy_has_valid_permission s2_policy_has_valid_permission))
```

### 混合 Effect 类型（Allow + Deny）

Deny 会覆盖 Allow。使用以下模式：
```smt2
(define-fun any_allow_valid () Bool (or s1_allow_valid ...))
(define-fun any_deny_valid () Bool (or s2_deny_valid ...))
(define-fun policy_has_valid_permission () Bool (and any_allow_valid (not any_deny_valid)))
```

### 第一步：分析

### 第二步：逐约束设计 SMT 方案
每个约束考虑：①定位 JSON 路径 ②设计 has_X + X_value 变量 ③设计验证逻辑

### 第三步：生成完整 SMT 代码
1. 声明所有变量（s1_has_effect, s1_effect_value 等）
2. 断言配置实际值（精确提取 IAM 配置，不能修改或编造）
3. 定义验证函数（有效值集合检查、条件兼容性检查）
4. 组合整体有效性 → assert → check-sat → exit

## 核心准则

1. **存在性+值分离**：每个字段有 `has_X`(Bool) 和 `X_value`(String)
2. **值验证必须用有效值集合**：`(or (= X "v1") (= X "v2") ...)`，不能只与自身比较
3. **Condition 须做操作符-键类型兼容性检查**
4. **Action 与 Condition 独立**：禁止添加交叉字段验证
5. **禁止 Z3 Set 类型**：只能用 declare-const + define-fun (or ...) 模式
6. **空 Condition {} 时设 has_condition=false**，不编造 operator/key/value
7. **仅编码实际存在的字段**，不存在的设 has_X=false
8. **键类型分类必须使用全部已知键**：key_is_string / key_is_numeric / key_is_bool 等函数必须列出所有已知键，不能只包含当前配置中的键
